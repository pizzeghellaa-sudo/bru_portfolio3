# bru_portfolio

Personal portfolio SSG for Bruna Bulgarelli (Brand & Visual Designer). Statically generated site with real per-locale URLs, server-rendered HTML on first load, and React hydration for interactivity.

## Tech Stack

- React 19 + TypeScript 5.8 + Vite 6.2
- **vite-react-ssg** — static site generation / prerendering (all routes prerendered at build time)
- **react-router-dom v6** — routing (v6.x required for vite-react-ssg compatibility)
- **react-helmet-async v1.x** — per-page head management (v1.x required; v3 breaks vite-react-ssg SSR context)
- Tailwind CSS v4 (via `@tailwindcss/vite` plugin — CSS-first config, no `tailwind.config.js`)
- Framer Motion (`motion` package, `motion/react` imports)
- Lucide React (icons)
- `@vercel/analytics` (Vercel Analytics)
- Node 18+

## Commands

- `npm run dev` — dev server at `http://localhost:3000`
- `npm run build` — regenerates `public/sitemap.xml`, then prerenders all routes via vite-react-ssg to `dist/`
- `npm run build:spa` — plain `vite build` escape hatch (skips SSG prerendering)
- `npm run preview` — preview production build
- `npm run clean` — delete `dist/`
- `npm run lint` — TypeScript type check (`tsc --noEmit`)

## Architecture

Each route is prerendered to a static HTML file at build time by `vite-react-ssg`, then hydrates into a normal SPA on the client. All 26 routes (10 section pages + 16 project detail pages) are emitted under `dist/`.

```
src/
  main.tsx              # vite-react-ssg entry — exports createRoot = ViteReactSSG({ routes })
  routes.tsx            # RouteObject[] consumed by vite-react-ssg + react-router-dom
  layouts/
    RootLayout.tsx      # Persistent shell: Sidebar + MobileMenu + AnimatePresence<Outlet> + global ImageModal
  sections/
    IndexSection.tsx
    TimelineSection.tsx
    CapabilitiesSection.tsx
    WorkSection.tsx
    ProjectDetail.tsx
    SignalSection.tsx
  components/
    Sidebar.tsx         # Desktop nav — uses NavLink, derives lang from useParams()
    MobileMenu.tsx      # Mobile drawer — uses NavLink, derives lang from useParams()
    ImageModal.tsx      # Zoom modal (cursor-centered zoom, 1×–3×)
    PageHead.tsx        # react-helmet-async wrapper — emits title, meta, canonical, hreflang, OG, JSON-LD
  i18n/
    useLanguage.ts      # Hook: useParams().lang → 'EN' | 'IT'
    localePaths.ts      # switchLocalePath(pathname, 'en'|'it') helper
  seo/
    siteMeta.ts         # SITE_URL, DEFAULT_OG_IMAGE, AUTHOR_NAME, SAME_AS
    pageJsonLd.ts       # buildPersonJsonLd, buildWebSiteJsonLd, buildProfilePageJsonLd, buildBreadcrumbJsonLd
    projectJsonLd.ts    # buildCreativeWorkJsonLd(project, lang, url)
  translations.ts       # EN/IT string map (TRANSLATIONS object)
  types.ts              # TypeScript interfaces + static data (EXPERIENCE[], PROJECTS[])
  lib/
    projectAssets.ts    # Dynamic asset loading via import.meta.glob
  index.css             # Tailwind v4 @theme config + global styles

scripts/
  generate-sitemap.mjs  # Regenerates public/sitemap.xml before every build (inline data, no tsx needed)

assets/
  bru.png
  projects/
    <project-id>/
      thumb.jpg         # Archive grid thumbnail (used if no video)
      <filename>.mp4    # Optional video thumbnail (overrides thumb.jpg in grid)
      <n>-thumb.jpg     # Gallery list thumbnails
      <n>-full.jpg      # Zoom modal high-res images

public/
  sitemap.xml           # Auto-generated by scripts/generate-sitemap.mjs at build time
  robots.txt
  llms.txt              # AI-crawler plain-text profile
  favicon.ico

vercel.json             # / → /en/ redirect (308), cleanUrls, trailingSlash
```

## Key Patterns

### Routing

`src/routes.tsx` defines a `RouteObject[]` consumed by vite-react-ssg. The `RootLayout` shell (sidebar, mobile menu, global image modal) stays mounted across route changes. Section transitions use `<AnimatePresence mode="wait">` keyed by `location.pathname` inside the `<Outlet>`. Every route is prerendered to static HTML at build time.

### Bilingual i18n

Language is URL-derived, not stateful. The `useLanguage()` hook in `src/i18n/useLanguage.ts` reads the `:lang` route param and returns `'EN' | 'IT'`. The language toggle calls `navigate(switchLocalePath(pathname, target))` from `src/i18n/localePaths.ts`. Translations still come from `TRANSLATIONS[language]` in `translations.ts`; bilingual content on data objects is accessed as `item.description[language]`.

### SEO & Head Management

Every page renders a `<PageHead>` (in `src/components/PageHead.tsx`) via `react-helmet-async`, producing per-route `<title>`, meta description, canonical URL, `hreflang` alternates (EN/IT/x-default), Open Graph, Twitter card, robots, and JSON-LD (Person, WebSite, ProfilePage, CreativeWork, BreadcrumbList as appropriate). vite-react-ssg automatically captures the Helmet output during prerendering and injects it into each route's `<head>`. JSON-LD builders live in `src/seo/`.

**Important:** react-helmet-async must stay at v1.x. v3 introduced a React 19 detection path that breaks the SSR context handshake with vite-react-ssg, causing Helmet output to land inside `#root` instead of `<head>`.

### Dynamic Asset Loading

`src/lib/projectAssets.ts` uses `import.meta.glob` to eagerly import all images and videos under `assets/projects/`. Three helpers:
- `projectThumb(id)` — returns `assets/projects/<id>/thumb.jpg`
- `projectGallery(id)` — returns `{ thumb, full }[]` pairs built from `<n>-thumb.jpg` / `<n>-full.jpg` naming convention
- `projectVideo(id, filename)` — returns `assets/projects/<id>/<filename>.mp4` (separate glob for `.mp4` files)

Each `Project` in `types.ts` now has a required `slug: string` field (kebab-case) used for URL construction. To add a new project: create `assets/projects/<id>/` with assets, add a `Project` entry in `types.ts` with a unique `slug`, and add the slug to `PROJECT_SLUGS` in `vite.config.ts` so it gets prerendered.

### Video Thumbnails

The `Project` interface has an optional `video?: string` field. When set, the Work grid renders a `<video>` element instead of `<img>` for that project card:
- Auto-plays (muted, no controls) with a **2-second delay** on first load via `onLoadedData` + `setTimeout`
- Loops with a **5-second pause** between iterations via `onEnded` + `setTimeout` + `video.currentTime = 0`
- No `autoPlay` or `loop` attributes — timing is fully controlled via event handlers

### Image Zoom Modal

`src/components/ImageModal.tsx` supports zoom from 1×–3× via:
- Mousewheel (cursor-centered: zoom anchors to the point under the cursor using `getBoundingClientRect` fractions + `requestAnimationFrame` scroll adjustment)
- `+`/`-` buttons and keyboard shortcuts
- Uses explicit pixel dimensions (`width: naturalWidth * zoom`) inside an `overflow: auto` scroll container — NOT CSS `transform: scale`

### Animations

Uses `motion` from `motion/react` (Framer Motion v12). `AnimatePresence mode="wait"` in `RootLayout` wraps section transitions. Standard pattern: `<motion.div initial={{ opacity: 0, y: 20 }} animate={{ opacity: 1, y: 0 }} exit={{ opacity: 0, y: -20 }}>`.

### Tailwind v4 Theme

Defined in `src/index.css` under `@theme {}` — no config file. Custom tokens:
- `--color-primary: #fa3d33`
- `--color-background-light: #f8f6f5`
- `--color-background-dark: #23100f`
- `--color-paper: #F4F4F0`
- `--color-ink: #000000`
- Fonts: `--font-display` (Work Sans), `--font-mono` (JetBrains Mono), `--font-montserrat` (Montserrat)

Custom utilities: `.swiss-grid-lines`, `.blend-multiply`, `.no-scrollbar`.

## Path Alias

`@/` maps to the project root (`.`). Configured in both `vite.config.ts` and `tsconfig.json`.

## Deployment

Deployed on **Vercel** with automatic deploys from the `main` branch. `vercel.json` handles the `/` → `/en/` redirect (308 permanent). `@vercel/analytics` is injected in `main.tsx`. The build command (`npm run build`) runs the sitemap generator before prerendering, so `dist/sitemap.xml` is always up-to-date.

## Notes

- Each route is prerendered as static HTML by `vite-react-ssg` during `npm run build`; inspect `dist/<lang>/<route>/index.html` to confirm SSG output
- react-router-dom is pinned to v6.x — vite-react-ssg 0.9.x is incompatible with RRD v7
- react-helmet-async is pinned to v1.x — see SEO & Head Management section for why
- `vite.config.ts` contains `ssgOptions.includedRoutes` with all 27 paths (26 content routes + root redirect). When adding a new project, update `PROJECT_SLUGS` there as well as `types.ts`
